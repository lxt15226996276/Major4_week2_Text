# 免费云端托管 — 关电脑也能用，你只负责维护

**你的需求完全可以满足：** 把站点上传到 Cloudflare / GitHub Pages 之后，**电脑关机、合盖、断网都不影响**同学访问。网站 24 小时在云端运行，你只在改题库时开机维护几分钟。

```
平时：电脑随便关 → 同学照常刷题（云端链接）
维护：改 Excel → 双击「上传云端.bat」→ 重新上传 dist（约 5 分钟）
```

> **`启动刷题站.bat` / `start.bat` 是临时方案**，必须保持电脑开机，适合你自己调试。  
> **给同学长期用，请走下面的云端部署，部署一次即可。**

---

刷题站本质是 **静态网页 + JSON 题库**（整个站点约几 MB），**不需要买云服务器**，也**不需要**在你电脑上 24 小时跑 Python。

| 方式 | 同学访问时你的电脑 | 内存占用 | 固定链接 | 费用 |
|------|-------------------|----------|----------|------|
| **`start.bat` 临时隧道** | 必须开着 | 本机跑服务 | 每次变 | 免费 |
| **GitHub / Cloudflare Pages** | **不用开** | **几乎为 0** | **永久** | **免费** |

部署到云端后：题库在 CDN 上，同学在浏览器里刷题，**算力和内存都在访问者手机/浏览器里**，不会把你电脑内存撑爆。

---

## 最省事：Cloudflare Pages（不用 Git）

适合：不想装 Git、只想拖文件夹上传。

### 第一次

1. 注册 [Cloudflare](https://dash.cloudflare.com/sign-up)（免费）
2. 双击 **`上传云端.bat`**（会先打包，再打开 `dist` 文件夹）
3. 打开 [Cloudflare Pages](https://dash.cloudflare.com/?to=/:account/workers-and-pages) → **Create** → **Pages** → **Upload assets**
4. 项目名称随意（如 `major4-quiz`），把 **`dist` 文件夹**拖进去上传
5. 部署完成后得到固定链接，例如：  
   `https://major4-quiz.pages.dev`

6. 把链接写入 `deploy.config.json` 的 `publicUrl`，再运行一次 **`上传云端.bat`** 并重新上传（分享页会显示正确地址）

### 更新题库

改 Excel → 双击 **`上传云端.bat`** → 在 Cloudflare 同一项目点 **Create deployment** 再传新的 `dist`。

## 其它免费平台（同样够用）

| 平台 | 特点 |
|------|------|
| [Netlify Drop](https://app.netlify.com/drop) | 拖 `dist` 文件夹，秒得链接 |
| [Vercel](https://vercel.com) | 连 GitHub 自动部署 |
| GitHub Pages | 已集成脚本，国内访问有时较慢 |

以上都是 **静态托管**，不是虚拟机，**没有「服务器内存爆炸」的问题**。

---

## 什么时候才需要「真·云服务器」？

只有这些情况才要考虑 VPS（阿里云/腾讯云等，通常不免费）：

- 必须在网页上**实时录入题库**且立刻全员可见（需要后端 + 数据库）
- 要做登录、统计后台、多人协作编辑

当前刷题功能 **不需要 VPS**。录入流程是：本地改 Excel → 打包 → 上传云端（几分钟一次即可）。
