## 二、场景清单

| 场景文件 | 存放路径 | 用途 |
|----------|----------|------|
| `game1.unity` | `Scenes/` | 开场：播放视频，5 秒后跳转 |
| `game2.unity` | `Scenes/` | 主玩法：战斗、怪物、门、回场景1 |

### game1 需要的物体

| 物体名 | 组件 | 关键参数 |
|--------|------|----------|
| Main Camera | Camera | Clear Flags: Solid Color；Background 随意 |
| Directional Light | Light | Intensity: 1 |
| VideoScreen | MeshRenderer + VideoPlayer | 见分步教程 Step 2 |
| SceneLoader（空物体） | SceneLoader 脚本 | 负责 5 秒切场景 |

### game2 需要的物体

| 物体名 | Tag/Layer | 组件 | 说明 |
|--------|-----------|------|------|
| Player | Player | CharacterController + Animator + AudioSource | 主角 |
| Main Camera | — | CameraFollow 脚本 | 跟随 Player |
| GameManager | — | GameManager 脚本 | 刷怪、胜负、BGM |
| Ground | — | MeshCollider | 地面 |
| DoorSpawnPoint | — | 空物体 Transform | 门生成位置（可选） |
| EventSystem | — | 若用 UI 则保留 | 本套可不强制 |

## 三、脚本清单

| 脚本名 | 挂载对象 | 职责 |
|--------|----------|------|
| `PlayerController.cs` | Player | WASD 移动旋转、Animator 参数、技能1冲刺 |
| `CameraFollow.cs` | Main Camera | 平滑跟随 Player |
| `MonsterController.cs` | 怪物预制体 | 待机、受击、死亡 |
| `SkillDash.cs` | Player（或合并进 PlayerController） | 按 1 键冲刺、伤害判定 |
| `GameManager.cs` | GameManager 空物体 | 随机刷 2~5 怪、计数、生成门 |
| `DoorTrigger.cs` | Door(Cube) | 触发特效音效、3 秒后 LoadScene game1 |
| `VideoSceneController.cs` | game1 空物体 | 播放视频、5 秒后 LoadScene game2 |

> **命名空间建议**：`namespace Exam01 { ... }`  
> **类名必须 PascalCase**，不能用中文或小写开头。

## 五、断网自检清单

- [ ] Build Settings 已添加 game1、game2，且 game1 在 index 0
- [ ] 所有资源引用无 Missing（断网后打开场景检查）
- [ ] 视频文件在 `Video/` 本地，Render Mode 正确
- [ ] 怪物预制体 Collider 不是 Trigger（或技能侧用 Trigger，规则要统一）
- [ ] 按 Play：game1 播视频 → 5 秒进 game2 → 打怪 → 出门 → 3 秒回 game1
- [ ] Console 无报错；类名/注释符合规范
